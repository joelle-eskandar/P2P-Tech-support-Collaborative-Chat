# P2P Tech-Support & Collaborative Chat  
 (Peer-to-Peer) نظام دردشة موزّع لدعم الفنيين والتعاون بينهم

1. وصف المشروع
مشروعنا هو نظام دردشة Peer-to-Peer (P2P) يهدف إلى توفير قناة تواصل مباشرة بين الفنيين دون الاعتماد على خادم مركزي.  
يعتمد النظام على Direct Sockets لإنشاء اتصال مباشر بين العقد (Nodes)، مما يسمح بتبادل الرسائل بسرعة وكفاءة، ويعمل كشبكة رديفة للنظام المركزي الخاص بالمجموعة المرآة.

هذا المشروع جزء من متطلبات مقرر برمجة التطبيقات الشبكية 

2. أهداف المشروع
- إنشاء شبكة رديفة تعمل حتى في حال توقف الخادم المركزي.
- توفير قناة تواصل مباشرة بين الفنيين لتبادل الخبرات.
- تقليل الضغط على الخادم المركزي عبر توزيع الحمل على العقد.
- تدريب الطلاب على بناء تطبيقات موزعة باستخدام نموذج P2P.
- استخدام GitHub في إدارة الإصدارات والعمل التعاوني.


3. المكونات الأساسية للنظام
- واجهة تواصل لإرسال واستقبال الرسائل.
- بروتوكول لاكتشاف الأقران (Peer Discovery).
- اتصال مباشر باستخدام TCP Sockets.
- إدارة جلسات اتصال بين العقد.
- إمكانية تشغيل التطبيق على Linux (Ubuntu) باستخدام GCC.


 4.بيئة التطوير
- لغة البرمجة: C  
- المترجم: GCC  
- نظام التشغيل: Ubuntu / Linux  
- أدوات إضافية: Git – GitHub


 5.العلاقة مع المجموعة المرآة
- المجموعة المرآة: Al-Powered Network Technical Support System (Client-Server)
- التكامل بين المشروعين:
  - هم يطوّرون نظام دعم فني مركزي.
  - نحن نطوّر شبكة رديفة موزعة تعمل عند تعطل الخادم.
- سيتم تبادل المستودعات لاحقًا لتحليل الفروقات بين النموذجين.
