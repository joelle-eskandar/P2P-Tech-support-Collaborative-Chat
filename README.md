# P2P Tech-Support & Collaborative Chat  
 (Peer-to-Peer) نظام دردشة موزّع لدعم الفنيين والتعاون بينهم

1. وصف المشروع
مشروعنا هو نظام دردشة Peer-to-Peer (P2P) يهدف إلى توفير قناة تواصل مباشرة بين الفنيين دون الاعتماد على خادم مركزي.  
يعتمد النظام على TCP Sockets لإنشاء اتصال مباشر بين العقد (Nodes)، مما يسمح بتبادل الرسائل بشكل مباشر بين الاطراف المتصلة ، ويعمل كشبكة رديفة للنظام المركزي الخاص بالمجموعة المرآة.

هذا المشروع جزء من متطلبات مقرر برمجة التطبيقات الشبكية 

2. أهداف المشروع
-إنشاء تطبيق دردشة يعتمد على نموذج Peer-to-Peer.
-توفير قناة تواصل مباشرة بين الفنيين.
-دراسة الفرق بين نموذج P2P ونموذج Client-Server.
-تطبيق مفاهيم TCP Sockets و Multithreading.
-استخدام Git و GitHub لإدارة الإصدارات والعمل الجماعي.

3. المكونات الأساسية للنظام
- إرسال واستقبال الرسائل بين الطرفين.
- اتصال مباشر باستخدام TCP Sockets.
- استخدام Threads للإرسال والاستقبال بشكل متزامن.
- إمكانية تشغيل التطبيق على Linux (Ubuntu) باستخدام GCC.


 4.بيئة التطوير
- لغة البرمجة: C  
- المترجم: GCC  
- نظام التشغيل: Ubuntu / Linux  
- أدوات إضافية: Git – GitHub


 5.العلاقة مع المجموعة المرآة
- المجموعة المرآة: AI-Powered Network Technical Support System (Client-Server)
- يقوم مشروع المجموعة المرآة على نموذج Client-Server باستخدام خادم مركزي، بينما يعتمد مشروعنا على الاتصال المباشر بين العقد دون الحاجة إلى خادم مركزي.
يسمح ذلك بدراسة الفروقات بين النموذجين من حيث الأداء، إدارة الاتصالات، والاعتمادية.
